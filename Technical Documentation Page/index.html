Comments

  • Single-line comments:

    Use the '#' symbol.

  • Multi-line comments: Use triple quotes (' ' ' or " " ")

"""
This is a multi-line comment
or docstring (more on this later).
"""

#This is a single-line comment

Docstrings

  • Docstrings are string literals that appear right after the definition of a function, method, class, or module. They are used to document the code.

  • Use triple quotes (''' or """).

# mymodule.py
def my_function():
"""
This is a docstring. It explains what the function does.
"""
pass

Module and Packages

  • A module is a single Python file that can contain functions, classes, and variables.

  • A package is a collection of modules in directories that provide a hierarchy of modules.

# mymodule.py
def my_function():
"""
Function in a module.
"""
pass

Standard Documentation Strings (Docstrings)

  • PEP 257 is the convention for writing docstrings in Python

  • Example for a function:

def add(a,b):
"""
Add two numbers.

Parameters:
a (int): The first number.
b (int): The second number.

Returns:
int: The sum of the two numbers.
"""
return a + b

Using Docstrings in REPL or ID

  • You can access the docstring of an object using the __doc__ attribute.

print(add._doc_)

Documentation Tools

  • Sphinx: A tool that makes it easy to create intelligent and beautiful documentation.

  • reStructuredText (reST): The default markup language used by Sphinx.

  • pydoc: A built-in module that can be used to view and generate documentation.

Writing Documentation with Sphinx

  • Install Sphinx:

    • pip install sphinx

  • Initialize Sphinx in your project:

    • sphinx-quickstart

  • Build the documentation:

    • make html

Best Practices

  • Be concise but clear.

  • Use proper grammar and punctuation.

  • Provide examples where appropriate.

  • Follow the conventions laid out in PEP 257.
